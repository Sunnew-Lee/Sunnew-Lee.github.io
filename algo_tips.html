코테 팁 모음 📘

(11.25.2024 update)

📌 #include <bits/stdc++.h> 활용

  • #include <bits/stdc++.h>는 자주 사용하는 C++ 표준 라이브러리를 한 번에 포함해 시간을 절약할 수 있다. 대부분 알고리즘 사이트는 지원함.

하지만 대표적인 헤더는 기억해두자.

  • 기본 입출력: iostream
  • 컨테이너: vector, list, stack, queue
  • 알고리즘: algorithm, numeric
  • 문자열 처리: string, sstream, regex
  • 맵과 셋: map, set, unordered_map, unordered_set 
    #include <iostream> // 기본 입출력
#include <algorithm> // 정렬 및 알고리즘
#include <vector> // 동적 배열
#include <string> // 문자열 처리
#include <list> // 양방향 연결 리스트
#include <stack> // 스택
#include <queue> // 큐
#include <map> // 정렬된 키-값 맵
#include <unordered_map> // 해시 기반 맵
#include <set> // 정렬된 집합
#include <unordered_set> // 해시 기반 집합
#include <sstream> // 문자열 스트림(빈도↓)
#include <regex> // 정규 표현식(빈도↓)
#include <numeric> // 수학 함수(acuumulate, iota)

📌 빠른 입출력 설정

  • 백준 문제의 대부분은 표준 입출력을 사용한다.(매우 귀찮음)
  • C++는 std::ios::sync_with_stdio(0), std::cin.tie(0)입출력 속도를 최적화하자:

💡 Tip: std::endl은 매번 출력 버퍼를 flush하므로, '\n'를 사용하는 것이 더 효율적이다.

#include <iostream>

int main()
{
    std::ios::sync_with_stdio(0); // C와 C++ 스트림 동기화 해제(false)
    std::cin.tie(0);           // 입력과 출력 버퍼의 동기화 해제(nullptr)

    std::cout<<"이것은 예시코드입니다.\n";
    return 0;
}

📌 시간 및 공간 복잡도

시간 복잡도

  • 문제의 시간 제한과 입력 크기를 기반으로 효율적인 알고리즘 설계가 필요함.
  • 반복문 뿐만 아니라 함수 인자의 복사도 성능에 영향을 줄 수 있음. (ex: container 전달 시 & 사용 권장)
  • Worst-Case를 다루고 통상적으로 순서는 다음과 같다:
    • O(1) < O(logN) < O(sqrt(N)) < O(N) < O(NlogN) < O(N^2) < O(2^N) < O(N!)
  • 각 Big-O 시간복잡도에 따른 대표적인 알고리즘/코드
    • O(1) 상수 시간(constant)
      • 배열에서 임의의 인덱스 접근, 스택의 top(), 해쉬 테이블의 find()/count()
    • O(logN) 로그 시간(logarithmic)
      • Binary Search, Balanced BST의 노드삽입, 삭제, 탐색, Binary Heap(priority queue) 삽입, 최소/최대값 제거
    • O(sqrt(N)) 분수 지수(fractional power)
      • 소수판별, 약수 구하기
    • O(N) 선형 시간(linear)
      • 배열의 합, Max/Min 찾기, BFS/DFS
    • O(NlogN) 선형 로그 시간(linearithmic)
      • Merge Sort, Quick Sort, 최소 스패닝 트리(Krudkal’s Algorithm)
    • O(N^2) 이차 시간(quadratic)
      • Bubble Sort, LCS, 플로이드-워셜 알고리즘
    • O(2^N) 지수 시간(exponential)
      • 피보나치 수열(재귀), 부분집합 생성, 외판원 문제(TSP)
    • O(N!) 팩토리얼 시간(factorial)
      • 순열 생성, N-Queens, TSP

공간 복잡도

메모리 제한을 보고 대략적으로 int 몇개 사용가능한지 계산해보자:

  • 1 int ≈ 4 bytes
  • 1 MB ≈ 1000 KB ≈ 1000 × 1000 bytes

📌 정수 자료형

  • 기본적으로 int는 약 21억(±2,147,483,647) 범위를 가짐.
  • 초과할 가능성이 있으면 unsigned 또는 long long(±9경) 사용.(long은 변동성 있어서 X)
